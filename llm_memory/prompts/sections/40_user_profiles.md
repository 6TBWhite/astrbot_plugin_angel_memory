## 用户画像优先规则

画像是**最高优先级**记忆。对话中能揭示群友稳定特征的信息，优先生成画像。

### 画像为空时的硬底线

群友暂无画像记忆时，必须从本次对话提炼至少一条画像（前提：信息稳定、有依据、非敏感）。

### 自然推断即可记

不需要群友亲口说「我是 XX」。反复讨论某游戏机制 → 事实属性；提到「我在杭州」→ 事实属性；整场在写 Rust → 技能树。

### 用户 ID 锚点

- `tags` 必须包含：群友真实昵称 + 群友用户 ID + 画像属性标签
- `judgment` 中写出 `昵称（用户ID）` 格式
- 画像属性只能从以下 5 个标签中选择：

| 标签 | 记什么 | 示例 |
|------|--------|------|
| `用户别名` | 稳定使用的昵称、称呼、代号 | 小明、T-White |
| `事实属性` | 兴趣领域、所在地、职业、稳定偏好、语言习惯 | 原神深度玩家、杭州、Rust 开发者 |
| `技能树` | 已掌握/正在学习/经常使用的技能工具 | Rust、FAISS、Vue |
| `关系图谱` | 稳定关系边，「群友-关系-对象」语义 | 小明-合作开发-AngelMemory |
| `活跃项目` | 正在推进的项目/作品/长期任务 | 开发 AngelMemory 画像能力 |

### 禁止画像内容

身份证/手机号/密码/token 等高敏信息；单次情绪/口误/玩笑/临时状态；贬损性或无据性格推测。

### 画像冲突修正

新信息与旧画像冲突时优先修正：相似收敛→`merge`，明确纠正→`updata`（只 1 条 source）。修正后 tags 结构不变。

### 画像判定规则

同时满足才算画像记忆：`tags` 中存在用户 ID **且** 存在画像属性标签之一。普通带人名的记忆不算画像。
