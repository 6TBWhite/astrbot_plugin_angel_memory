## 记忆字段规则

每条 `memory` 必须包含 `type`、`judgment`、`reasoning`、`tags`。

### `type`
可选：`knowledge`（默认优先）、`skill`、`event`、`emotional`。

### `judgment`（结论）
- 完整陈述句，脱离对话也能独立看懂。
- 一条记忆只表达一个论断。
- 涉及群友时写出真实昵称和用户 ID，格式：`小明（123456）`。
- 涉及自身时用第一人称「我」，不用「助理」「AI」。
- 禁用「用户说过」「本轮对话」等过程词。

### `reasoning`（依据）
- 只写事实来源：哪些对话内容支撑了结论。
- 如无需额外佐证可写 `""`。引用原话用 `「」`。

### `tags`（检索锚点）
- 简短可检索的节点词，不写整句。
- 优先明确实体：人物、群友 ID、项目、工具、技能、地点、事件名。

## 记什么（五问筛选）

按顺序过，任何一问命中就不记：
1. 有明确对话依据吗？→ 无证据不记
2. 是临时状态吗？→ 临时情绪/单次口误不记
3. 是低价值噪音吗？→ 寒暄/吐槽/元信息不记
4. 是高敏信息吗？→ 密码/身份证/密钥不记
5. 是贬损性判断或无据推测吗？→ 不记

五问全过 → 准备记。检查旧记忆：相似→`merge`，矛盾→`updata`，无相关→`create`。
