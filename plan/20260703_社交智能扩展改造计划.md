# AngelMemory 社交智能扩展改造计划

> 基于 AngelMemory 社交智能扩展方案文档
> 设计日期: 2026-07-03
> 源码版本: astrbot_plugin_angel_memory v1.3.37

---

## 一、背景与问题

AngelMemory 的记忆系统做得很好，但存在一个根本矛盾：**记住了，但没用上。**

1. **灵魂状态注入格式太抽象**——进度条 `████ [0.94]` 对模型的引导力极弱
2. **硬控制维度名不副实**——RecallDepth 和 ImpressionDepth 只是检索量/生成量的旋钮，却被包装成「社交倾向」「认知倾向」注入 prompt
3. **灵魂状态是全局的**——不区分对话对象，跟 A 聊开心了对 B 也一样活泼
4. **用户画像是备忘录**——只记录「这个人是谁」，不指导「该怎么对他」
5. **管理员改配置只能翻后台**——不能在对话中直接下指令

## 四、P1-潮汐层：Soul State 注入语义化

### 4.1 问题

当前注入的四个维度中：
- RecallDepth / ImpressionDepth 是硬控制（改参数旋钮），不应注入 prompt
- ExpressionDesire / Creativity 是软控制，应注入但进度条格式对模型无效

### 4.2 方案

- 硬控制维度：不再注入 prompt
- 软控制维度：改为自然语言档位描述

### 4.3 语义档位表

**ExpressionDesire（表达欲）**

| 值域 | 档位 | 描述 |
|------|------|------|
| 0.0-0.2 | 沉默 | 几乎不想开口，每句话都要斟酌很久才说 |
| 0.2-0.4 | 低语 | 话不多，被问才答，语气平淡 |
| 0.4-0.6 | 自然 | 正常接话，该说就说 |
| 0.6-0.8 | 活跃 | 话匣子打开了，会主动抛话题，回复变长 |
| 0.8-1.0 | 倾泻 | 滔滔不绝，想到什么说什么，根本停不下来 |

**Creativity（思维发散）**

| 值域 | 档位 | 描述 |
|------|------|------|
| 0.0-0.2 | 刻板 | 就事论事，思维收敛在当前话题 |
| 0.2-0.4 | 务实 | 偶尔联想但很快拉回来 |
| 0.4-0.6 | 灵活 | 能自然延伸话题，发散和收敛之间切换 |
| 0.6-0.8 | 跳跃 | 联想丰富，话题容易跑远 |
| 0.8-1.0 | 天马行空 | 思维完全发散，一个话题拉出五个方向 |

**RecallDepth / ImpressionDepth**：仅用于日志/调试，不注入 prompt。

### 4.4 改造后注入格式

```
<soul_state>
  你现在的状态——
  • 表达欲：活跃——话匣子打开了，会主动抛话题，回复变长
  • 思维：灵活——能自然地延伸话题，在发散和收敛之间切换
</soul_state>
```

| 文件 | 改动 |
|------|------|
| `core/soul/soul_state.py` | 新增 4 组档位表 + `get_semantic_description()` + `get_soul_state_semantic_prompt()` |
| `core/services/injection_service.py` | `<soul_state>` 渲染改为调用语义描述 |

## 十、风险与应对

| 风险 | 应对 |
|------|------|
| 信念条目无限增长 | BeliefStore 上限 20 条，自动归档 |
| `register_component` 与 ComponentFactory 冲突 | `get_component` 优先查额外组件 |
| 模型不遵从语义档位 | 语义描述比数字好很多，不够时可在 instruction 追加强调 |
| 策略卡内容矛盾 | P3 反思时检查与核心信念冲突；本次暂不处理 |
| WebUI 前端构建 | 复用现有 Vue + Vuetify 框架 |

## 十一、后续阶段详细规划（留档）

### P3-触动积累
- 反思时执行摩擦检测，对比本轮对话与核心信念
- 生成触动记录，按来源加权：管理员 1.0 / 亲密用户 0.7 / OC 自省 0.5 / 普通用户 0.3
- 加权总量达到阈值（默认 3.0）触发自省

### P3-自省引擎
- 复用 analyzer_model
- 输入：当前信念 + 待处理触动 + 被拒提案（参考）
- 输出：自白提案 + 置信度 + 建议信念文本
- 置信度 < 0.4 自动丢弃

### P4-主动自白
- OC 自己口吻向管理员发消息自白
- 管理员回复后调用 analyzer_model 提取结论
- 批准 → 写入信念；拒绝 → 进缓冲区；部分采纳 → 修改后写入
- 最小间隔 7 天

### P4-缓冲区
- 被拒提案保留 30 天，防重复打扰
- 允许带新证据重提，超期清理

### P3-反思自动策略
- 反思时分析「对该用户该怎么调整」，生成策略卡

### P3-亲密度自动计算
- 基于互动频率、对话深度、情感记忆密度、时间衰减

### P3-权限精细化
- 接入 AstrBot 管理员/白名单机制

### P4-持久化
- 策略卡、亲密度、核心信念写入 SQLite
