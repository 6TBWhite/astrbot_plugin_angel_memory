## 待开发

### 1. 灵魂状态可调参数

**问题**：橡皮筋的 k=0.3（敏感度）、弹性=0.1（偏离阻力）、回归=0.1（归中速度）、反思力度=1.0 全是硬编码，无法调整 OC 的情绪反应速度。

**方案**：
- 在 `_conf_schema.json` 的 `enable_soul_system` 下新增子项：
  - `elasticity_k`：敏感度系数（默认 0.3，越大越敏感）
  - `regression_factor`：归中速度（默认 0.1，越大回弹越快）
  - `reflect_strength`：反思调整力度（默认 1.0）
- `soul_state.py` 从 config 读取这些值

**工程量**：⭐ 小

---

### 2. 画像加速生成

**问题**：用户画像一直是空的，画像需要反思系统慢慢攒 `profile_attribute:` 标签的记忆，过程太慢。

**方案**：
- 在 `organize_and_inject_memories` 中，当当前会话用户**没有任何画像记忆**时，追加一段 prompt 指令，让反思 LLM 优先从本轮对话中提炼基础画像
- 或者在首次跟某人聊天时，主动让 LLM 推断 2-3 条基础画像（用户别名、事实属性）
- 需要修改 `_build_strategy_analysis_prompt` 或反思 prompt 的画像优先级部分

**工程量**：⭐⭐ 中小

---

### 3. 策略 scope 隔离

**问题**：用户使用人格隔离（Parameter_memory 域），但策略目前是全局的。在 A 人格下对某人的策略不应该影响 B 人格。

**上次尝试**：实现过 scope 隔离但导致 WebUI 查询复杂化，已回退。

**方案**：
- `StrategyStore` 存储 key 改为 `scope:user_id`
- 注入时按当前 scope 查找，找不到回退 public
- WebUI 查询时增加 scope 选择器（或默认查 public + 当前活跃 scope）
- 工具写入时自动获取当前 scope

**工程量**：⭐⭐ 中小

---

### 4. WebUI 灵魂仪表盘

**问题**：灵魂状态四条能量槽只能看日志的数字，没有可视化。

**方案**：
- 新建 `SoulStateView.vue`
- 实时展示四维能量值（进度条 + 数值）
- 可选：历史变化曲线（需要后端记录能量值变化）
- 在 `routes.py` 新增 `/soul_state` API

**工程量**：⭐⭐ 中小

---

### 5. 主动自白（P4-暂缓项）

**问题**：自省提案通过后只能暂存，OC 不会主动找管理员讨论。

**依赖**：AstrBot 的主动消息能力（proactive task / 主动发消息）

**方案**：
- 自省提案通过后，OC 用自己的口吻主动向管理员发消息
- 管理员回复后，调用 analyzer_model 提取结论（批准/拒绝/部分采纳）
- 批准 → 调用 `angel_self_evolve` 写入信念
- 拒绝 → 进入缓冲区
- 最小间隔 7 天，冷却期内不发第二次
- 需要调研 AstrBot 的主动消息 API

**工程量**：⭐⭐⭐⭐ 中大
