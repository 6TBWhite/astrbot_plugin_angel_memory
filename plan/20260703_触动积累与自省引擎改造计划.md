# 触动积累 & 自省引擎 改造计划

> 基于 AngelMemory 社交智能扩展方案 P3 阶段
> 设计日期: 2026-07-03
> 源码版本: astrbot_plugin_angel_memory v1.3.37

---

## 一、背景

当前核心信念只能由管理员手动写入或 WebUI 编辑。OC 在聊天中积累了经验但无法自主形成「我觉得我可能需要调整」的认知。需要让 OC 能够：

1. 察觉对话中与自身信念有摩擦的信息（触动）
2. 攒够了触动后主动自省，形成自白提案

## 二、目标

| 编号 | 内容 | 说明 |
|------|------|------|
| P3-触动积累 | 反思时检测与核心信念有摩擦的信息，生成触动记录，按来源加权累积 | 新建 |
| P3-自省引擎 | 触动加权总量达到阈值时触发自省，调用分析模型生成自白提案 + 置信度 | 新建 |

## 三、非目标（本次不做）

## 四、触动积累

### 4.1 触发时机

每次反思完成后（与策略自动更新相同位置），追加一步「信念摩擦检测」。

### 4.2 触动记录结构

```json
{
  "id": "impulse_20260703_001",
  "content": "三个不同用户都反馈我说话太多",
  "direction": "表达欲需要在不熟悉的人面前收敛",
  "source_memories": ["mem_id_1", "mem_id_2"],
  "source_users": ["1432192649"],
  "trust_weight": 0.6,
  "created_at": 1751558400
}
```

### 4.3 信任权重

| 来源 | 权重 |
|------|------|
| 管理员反馈 | 1.0 |
| 多个独立用户同类反馈 | 0.7 |
| OC 自己反思发现的矛盾 | 0.5 |
| 单一普通用户反馈 | 0.3 |

### 4.4 生成方式

- 复用反思的 LLM 响应，在 prompt 中追加信念摩擦检测指令
- LLM 返回冲动列表 `"impulses": [...]` 
- 每个冲动包含 content、direction、trust_weight

### 4.5 存储

- 新建 `ImpulseStore`，注册为额外组件，持久化 JSON
- 提供 `add_impulse()` / `get_pending_total_weight()` 方法

## 五、自省引擎

### 5.1 触发条件

```
sum(pending_impulse.trust_weight) >= threshold (默认 3.0)
```

举例：
- 3 个普通用户各说一次（3 × 0.3 = 0.9）→ 不触发
- 3 个普通 + 管理员 1 次（0.9 + 1.0 = 1.9）→ 不触发
- 管理员 3 次（3 × 1.0 = 3.0）→ 触发

### 5.2 自省流程

```
触动累积达标
  ↓
调用 analyzer_model 进行自省
输入：当前核心信念列表 + 所有待处理触动
输出：自白提案 + 置信度评分
  ↓
置信度 < 0.4 → 自动丢弃，记录日志
置信度 ≥ 0.4 → 暂存为「待自白」状态
```

### 5.3 自白提案结构

```json
{
  "proposal": "我想在不太熟的人面前控制自己的表达欲",
  "reasoning": "最近有3个人反馈我话太多，自己反思也发现对不熟的人展开太多话题会让对方不舒服",
  "confidence": 0.72,
  "suggested_belief_text": "对不太熟悉的人聊天时，先观察对方的节奏，不要急着展开话题"
}
```

### 5.4 待自白状态

- 提案通过置信度检查后，暂存到 ImpulseStore 的 `pending_confessions` 列表
- 日志输出 `[自省] 提案通过 session=xxx confidence=0.72`
- **本次不自动发消息**（留 P4），暂时通过日志和 WebUI 可查看
- 下次继续触发时可以合并同类提案

## 六、与反思管线的整合

在 `_execute_async_analysis_task` 中，已有反思 → 灵魂更新 → 策略自动 → 亲密度自动。新增步骤：

```
反思
  → 灵魂状态更新
  → 记忆生成
  → 策略自动更新
  → 亲密度自动计算
  → ⭐ 信念摩擦检测（触动生成）  ← 新增
  → ⭐ 自省触发检查               ← 新增
```
